# navi_ota課題作成

navi_otaプロジェクトに以下の固定設定で課題を作成してください：

## 固定フィールド

- **プロジェクト:** navi_ota
- **親タスク:** navi_ota-15
- **状態:** backlog
- **責務:** フロントエンド（※配列で渡す: `["フロントエンド"]`）
- **タイプ:** User Story（※フィールド名は `タイプ` を使用）
- **UNIT:** OTA PJ

## カスタムフィールドの正しい形式

```json
{
  "State": "backlog",
  "責務": ["フロントエンド"],
  "タイプ": "User Story",
  "UNIT": "OTA PJ"
}
```

**注意:**

- `責務` は配列型（複数選択可能なため）
- `Type` ではなく `タイプ` を使用

## ユーザー入力

$ARGUMENTS

## 作成手順

1. ユーザー入力からタイトルを整形
2. ユーザー入力から受け入れ条件を抽出し、以下の形式で説明を作成：

```
## 受け入れ条件
- [ ] 〜できる
- [ ] 〜していること
```

3. youtrack mcpを使用して課題を作成（上記のカスタムフィールド形式を使用）
4. 作成結果（ID、URL、課題のタイトル、内容）を表示
